# Data Safety

本文件用于约束论文正式录用前的求职展示仓库公开范围。目标是在不公开完整数据、权重、投稿材料和核心实验仓库的前提下，保留项目说明、最小推理入口和面试展示材料。

## 1. 公开仓库可以包含什么

可以包含：

- `README.md`
- `requirements.txt`
- `.gitignore`
- 单图推理脚本，例如 `scripts/infer_image.py`
- FastAPI / Vue 的展示版工程代码
- 脱敏后的少量 demo 图片，命名为 `sample_001.jpg`、`sample_002.jpg`
- 脱敏后的少量 demo 输出图
- 不含敏感路径的实验结果摘要
- 不含投稿细节的面试讲解稿
- 不含完整日志的工程化计划

## 2. 公开仓库不能包含什么

不能包含：

- 完整 `datasets/`
- 完整 `runs/`
- 完整 `weights/`
- 完整 `paper/`
- `.pt`、`.pth`、`.onnx`、`.engine` 权重或导出模型
- 论文 `.docx`、`.pdf`
- cover letter、declarations、投稿系统上传包
- 审计报告、投稿前检查报告、内部复核材料
- 完整测试集图片
- 原始编号暴露的数据图片，例如直接使用原测试集编号作为文件名
- 含有本地绝对路径、个人路径或投稿目录结构的文档

## 3. 面试时可以展示什么

可以展示：

- 项目背景和任务定义
- CE7-DET 的类别缩写和检测任务形式
- 少量脱敏 demo 图片和检测输出
- README 中的实验结果摘要
- ROI refinement 的思路示意
- 单图推理脚本的输入输出
- FastAPI / Vue 的工程化计划
- 自己负责的数据、训练、评估、可视化和展示版整理工作

## 4. 面试时不建议发送什么

不建议直接发送：

- 完整数据集
- 完整权重文件
- 完整 `runs/` 实验目录
- 论文投稿稿件
- 投稿材料压缩包
- 审计报告
- 包含全部预测文本、全部测试集图片或内部路径的文件
- 尚未脱敏的原始实验仓库

如果面试官需要代码，建议发送展示版仓库链接，而不是核心论文实验仓库。

## 5. 论文录用前的保护原则

- 只公开能说明项目能力的最小工程骨架。
- 只公开少量脱敏 demo asset。
- 指标只写摘要，不公开完整实验日志和投稿包。
- 权重和完整数据集只在本地保留。
- README 不暴露本地绝对路径。
- 不把项目包装成已工业部署或 SOTA。
- 不公开可能影响论文投稿、复审或版权的数据材料。

## 6. 后续开源原则

如果之后要开源，应等论文接收后再评估开源范围。

建议按以下顺序评估：

1. 数据集授权是否允许公开。
2. 权重是否允许发布。
3. 论文图表是否已正式发表。
4. 实验日志是否需要清理或脱敏。
5. 是否只发布复现脚本和配置，而不发布原始数据。
6. 是否将论文实验仓库和求职展示仓库保持分离。

论文接收前，推荐只公开展示版工程，不公开完整实验仓库。
